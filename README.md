# ドットでダービー！

ドット絵の競馬を題材にした、
短時間周回型の育成シミュレーションゲームです。
※ドットアニメーションは未実装

Unityを用いて、育成システムの基礎構造と、
仕様設計から実装までの一連の流れを理解することを目的に制作しました。


## 概要
プレイヤーは1頭の馬を育成し、
全36ターンの育成期間の中で行動を選択しながらステータスを成長させます。

12ターン・24ターン・36ターンの節目には目標レースが設定されており、
育成結果がそのままレース結果としてフィードバックされます。

最終レース終了後は育成結果を評価し、
因子として次の周回に一部ステータスを引き継ぐことで、
短時間・高回転の周回プレイを可能にしています。


## ゲームサイクル
1周の基本的な流れは以下の通りです。

1. 育成開始（全36ターン）
2. 各ターンで行動を選択（練習 / 休憩 / おでかけ）
3. 12・24・36ターン目に目標レースを実行
4. 最終レース後に結果・評価を表示
5. 因子抽選を行い、次の周回へ

1周あたりの想定プレイ時間は、約10〜15分です。


## 育成システム
### ステータス
育成対象には以下の5種類のステータスが存在します。

- スピード
- スタミナ
- パワー
- 根性
- 賢さ

補助パラメータとして以下を管理します。

- 体力（0〜100）
- やる気（0〜4段階）

### 行動コマンド
-「練習」
  ランダムなステータスが上昇します。
  体力ややる気に応じて成功・失敗率が変動します。

-「休憩」
  体力を回復します。一定確率でやる気が上昇します。

-「おでかけ」
  やる気回復・体力回復・ステータス上昇など、
  ランダムな効果が発生します。


## レースシステム
目標レースでは、育成したステータスをもとに
簡易的な計算式で勝敗判定を行います。

- ステータス合計に各種係数を掛けた総合力を算出
- やる気による補正
- ランダム補正

これらを加味した最終値によって、
勝利・敗北が決定されます。

レースは短時間のドットアニメーションで演出し、
結果が即座に表示されます。
※本バージョンではアニメーション未実装。


## 周回要素（因子システム）
育成終了後、結果に応じて因子抽選を行います。

- 最大3枠の因子を獲得
- 因子はステータス値として次周回に引き継がれる
- 抽選は1周につき1回のみ

これにより、周回を重ねるほど育成効率が向上し、
プレイヤーに成長の手応えを与える設計としています。


## 操作方法
- マウス操作のみでプレイ可能
- 各画面のボタンをクリックして進行
- レース画面は画面タップ（クリック）で続行


## 制作目的
本作は、ゲーム企画者として以下の点を、
実装を通して体感的に理解することを目的に制作しました。

- 育成仕様と数値設計の関係性
- ターン制進行における状態管理
- UI更新とゲームロジックの分離
- 仕様変更時の実装コスト感覚

「育成 → レース → 結果 → 因子 → 次周回」という  
コア体験が1サイクルとして成立することを最優先に設計しています。
